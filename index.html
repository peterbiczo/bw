Lazy loading

When working on website with a lot of media or iframe content it's important to load lazily. Meaning only load contents when those are near to what you are looking at. This has been introduced with the recent web update on the sport and the casino side as well. There is nothing wrong with lazy loading from the testing perspective, however if the tags are not in place it makes things complicated, more time-consuming and sensitive to changes.

Conclusion


If an element has an id it is supper easy to locate and control everything else within the test. When a website is under develepment it is essential to use tags as specific as possible. Once the item has an id, the content can be moved around, change size, apply lazy loading, etc... On the other hand when you don't have an id every time the classname or position is changed the scriptis going to have an error.
